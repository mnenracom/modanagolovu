# Решение проблем с WildBerries API

## Ошибка 404 "path not found"

Если вы получаете ошибку 404 при попытке синхронизации, это означает, что:

### Возможные причины:

1. **Токен не имеет доступа к Statistics API**
   - Убедитесь, что токен создан для **Statistics API** (статистика и финансы)
   - Не используйте токен для Marketplace API - это разные API

2. **Токен не активирован или истек**
   - Проверьте срок действия токена в личном кабинете WB
   - Создайте новый токен, если старый истек

3. **Endpoint недоступен для вашего токена**
   - Некоторые endpoints требуют специальных прав доступа
   - Обратитесь в поддержку WB для получения доступа

### Что проверить:

1. **В личном кабинете WildBerries:**
   - Перейдите в "Настройки" → "Доступ к API"
   - Убедитесь, что создан токен для **Statistics API**
   - Проверьте, что токен активен и не истек

2. **Проверьте тип токена:**
   - Statistics API использует JWT токены (начинаются с `eyJ...`)
   - Marketplace API использует другие типы токенов

3. **Проверьте доступ к endpoints:**
   - Попробуйте использовать токен в n8n или Postman
   - Если работает в n8n, но не работает здесь - проблема в формате запроса
   - Если не работает нигде - проблема в токене или доступе

### Альтернативные решения:

1. **Используйте другой endpoint:**
   - Попробуйте `/api/v1/supplier/reportDetailByPeriod` вместо `/api/v1/supplier/report`
   - Или наоборот

2. **Проверьте параметры запроса:**
   - Убедитесь, что даты в правильном формате (YYYY-MM-DD)
   - Проверьте, что все обязательные параметры переданы

3. **Обратитесь в поддержку WB:**
   - Создайте тикет в личном кабинете WB
   - Укажите, что получаете 404 при обращении к Statistics API
   - Приложите requestId из ошибки (например: `b764ffc3a6766526e291025c7148bd3e`)
